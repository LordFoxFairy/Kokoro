# 操作教训 · 仿参考设计前，先亲自打开参考站看透

## 场景
要把 Kokoro 的"工作室"做成像 muapi/pollo 那样的"功能组件"。用户给了截图和网址。

## 我做错的
没先深入打开 muapi/pollo 的 studio 页看清结构，就凭印象动手——连续 3 版都错：
1. 把"输入表单"放进**右侧产物区**（用户：右边是产物，输入不该在右）。
2. 改成"输入参数在左 composer + 右侧产物空态"的**左右分栏**（用户：也一般，没深入看别人怎么设计）。
3. 真去打开 muapi `/studio/video` 截全图，才看清它是**单列生成器**：身份大标题居中 + 中部留白/模板 + **底部全宽生成栏**（上传+输入 / 参数 chips + 生成），根本没有左右分栏。第 3 版才对。

每错一版都派了子 agent 往 13 个页铺，两次中途 STOP，浪费两轮。

## 下次怎么避免
- **仿谁的设计，先用 Playwright 把那个页面（含 full-page）截下来亲自看透**，列出它的真实结构（区块/控件/布局），再动手。别拿别人给的一句话或半张图就开干。
- 大改版（影响 N 个页）**先把 1 个锚样板做对、给用户确认，再铺量**。锚没确认就批量派工 = 错了 ×N。
- 用户说"参考对应网站""你应该知道如何处理" = 让我**对着真实参考做**，不是让我猜。
