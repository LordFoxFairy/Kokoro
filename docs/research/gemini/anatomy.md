# Gemini 界面拆解（anatomy）

> 基于用户提供的两张登录后产品截图：`screenshots/01-loggedin-home.png`（无 Canvas）与 `screenshots/02-loggedin-canvas.png`（开启 Canvas）；以及补抓的 `screenshots/03-loggedout-landing.png`（未登录态首页）作为对照。所有数值为目测估算，非源码精确值；以 Google Material Design 3 体系为旁证。
> 角色定位：Kokoro 的主轴参考。

---

## 1. 总体布局

### 1.1 默认态（无 Canvas）

```
┌────────────────────────┬─────────────────────────────────────────────┐
│  Sidebar  (~280px)     │  Main  (剩余宽度，淡蓝→白 渐变)               │
│                        │                                              │
│  - 品牌 / 折叠按钮       │                                              │
│  - 操作按钮组           │           大字问候语（垂直 ~1/3 处）           │
│  - 笔记本 section       │                                              │
│  - 最近 section（列表）  │           胶囊输入框                          │
│  - 用户区（底部固定）    │                                              │
│                        │                              悬浮工具（右下）  │
└────────────────────────┴─────────────────────────────────────────────┘
```

- 主区域为**非常淡的蓝→白渐变**（约 `#F0F4F9 → #FFFFFF`，左上向右下）
- 导航与主区域**无明显分割线**，靠背景色差 + 留白区分
- 主区域内容**水平居中**、**垂直偏上 1/3**

### 1.2 Canvas 态

```
┌──────────────┬─────────────────────┬───────────────────────────────┐
│  Sidebar     │  Chat               │  Canvas（文档面板）             │
│  (~240px,    │  (~中等宽度，白底)    │  (白底)                         │
│   变窄)      │                     │  顶部工具条 / 正文 / 右缘工具列     │
└──────────────┴─────────────────────┴───────────────────────────────┘
```

- Canvas 打开后，Sidebar **显著变窄**（视觉权重降级）
- Chat 与 Canvas 之间无可见分隔，靠 surface 阴影微差
- 中央栏顶部有"关闭边栏"chip 按钮，深色填充，浮在 Chat 顶端

## 2. 左侧导航（Sidebar）

### 2.1 结构

| 区段 | 内容 | 说明 |
|---|---|---|
| 品牌 | Gemini logo（彩色钻石）+ 折叠按钮 | 顶部，紧凑 |
| 主操作 | 发起新对话 / 搜索对话内容 / 库 / Gem | 4 个 item，图标+文字 |
| 笔记本 | 标题 + "+ 新建笔记本" | 小区段 |
| 最近 | 标题 + 历史对话列表 | 可滚动主体 |
| 用户区 | 头像 + 名字 + Pro 标 + 齿轮 | 底部固定 |

### 2.2 视觉细节

- **激活态**：当前选中（"发起新对话"）淡蓝填充 `#E8F0FE` 印象，圆角 ≈ 12px
- **图标**：Material Outlined 线性风格，描边约 1.5px
- **字号**：item 文字 ≈ 14px，分组标题 ≈ 12px 灰色
- **行高**：item 高度 ≈ 40px，左右 padding ≈ 12px
- **历史列表**：纯文本、无图标、无时间戳，行高 ≈ 32-36px
- **滚动条**：默认隐藏，hover 显示（推测）

## 3. 中央主区 — 默认态

### 3.1 问候语

- 字号 ≈ 36-44px，字重常规/中等（非 bold）
- 颜色灰调 `#3C4043` 印象
- 字体 Google Sans / Product Sans 印象
- 位置：垂直 1/3，水平居中

### 3.2 胶囊输入框

- 圆角 ≈ 24px（胶囊），高度 ≈ 56-64px
- 白底 + 极淡阴影
- 内部：`[+]  问问 Gemini ……………  [Flash ▾]  [🎤]`
- 左 + 号：圆形按钮，灰色细描边
- 模型切换：Pill chip，淡灰底，下拉箭头
- 麦克风：圆形按钮，灰色

### 3.3 右下角悬浮按钮

- 两个圆形按钮纵向堆叠，淡灰描边
- 推测：扩展工具 / 语言切换

---

## 4. 中央主区 — Canvas 态对话流

### 4.1 用户消息

- 淡灰胶囊气泡 `#F1F3F4` 印象，大圆角
- **水平居中**（非右对齐！Gemini 的辨识特征）

### 4.2 助手回复

- **无气泡**，纯文本左对齐
- 段落间距宽松

### 4.3 文档生成卡片

- 大圆角矩形 ≈ 12px，淡灰边框
- 左侧文档 emoji + 标题 + 状态行（"正在生成..."）
- 宽度小于消息宽度

### 4.4 输入框（Canvas 态）

- 多一个 **Canvas 切换按钮**（带 Canvas 图标 + 文字，淡蓝填充表示激活）
- 右侧麦克风替换为**圆形发送按钮**

---

## 5. 右侧 Canvas 面板

### 5.1 顶部工具栏

- 左：项目名 chip `项目...`
- 中：图标组（云同步 / 撤销 / 重做 / 更多 / 公式 / 打印 / 分享）
- 右：`创建 ▾`（深色填充主按钮）+ `×` 关闭

### 5.2 文档正文

- 标题：粗壮 sans-serif 或重 weight serif
- 正文：sans-serif
- 层级 H1 ≈ 28px / H2 ≈ 22px / H3 ≈ 18px
- 列表 bullet 圆点，缩进 ≈ 24px
- 文档信息块：定义列表样式（label 加粗 + value）

### 5.3 右缘垂直工具条

- 3 个图标纵向排列
- 推测：大纲 / 评论 / 翻译

## 6. 全局设计 tokens（印象值）

| Token | 估算 | 备注 |
|---|---|---|
| 主色 | `#4285F4` | Google Blue，仅极少使用 |
| 强调淡填充 | `#E8F0FE` | 激活态、Canvas 按钮 |
| 文本主 | `#1F1F1F` / `#202124` | Material on-surface |
| 文本次 | `#5F6368` | 占位、分组标题 |
| 边框/分割 | `#E0E0E0` | 极淡，几乎不用 |
| 主背景渐变 | `#F0F4F9 → #FFFFFF` | 左上→右下 |
| 圆角 大 | 24px | 输入框胶囊 |
| 圆角 中 | 12-16px | 导航 item / 卡片 |
| 圆角 小 | 6-8px | chip |
| 字体 | Google Sans 系 | sans-serif，几何 |
| 阴影 | 极轻 | `0 1px 6px rgba(0,0,0,0.04)` 印象 |

## 7. 设计语言总结

1. **极简留白** — 单页信息密度低，留白占比 > 50%
2. **柔和淡蓝** — 唯一品牌色，其余靠灰阶
3. **无强分割** — 靠 surface 色差 + 留白，几乎不用线
4. **大圆角** — 胶囊型输入，大圆角卡片，整体亲和
5. **图标克制** — 仅必要项，线性轻量
6. **字体优雅** — Google Sans，问候语大字号制造焦点
7. **状态柔和** — 激活淡填充，不用强对比

## 8. Kokoro 可直接吸收的点

1. **两栏 → 三栏**的基础布局（Canvas 时主区域横向再分）
2. **大圆角胶囊输入框** + 内部组件分布（左附件、中输入、右模型切换、右发送）
3. **历史对话列表纯文本无图标** 的极简列法
4. **主区域非纯白**，用极淡背景 / 渐变 区分功能区
5. **助手无气泡的叙述流**（保留：让 Kokoro 的回复"像一封信，不像一条消息"，是有意义的克制）
   ~~原 5：用户气泡居中~~ —— 已撤回，见 [ADR-008](../../decisions/ADR-008-chat-alignment.md)。用户气泡走业界标准右对齐，居中是 Gemini 凭用户基数撑得起的偏离，Kokoro 早期不该硬学。

---

## 9. 可能差异化的点（不无脑学）

1. Gemini 的"大字问候"很 Google 范，Kokoro 可考虑替换为更具产品个性的引导（如任务建议卡片、最近上下文摘要）
2. Canvas 顶部工具条**偏多偏密**，Kokoro 可考虑分组折叠
3. 整体偏冷色，Kokoro 可保留主色调克制但加入一抹**辨识度强的点缀色**
4. Sidebar 用户区底部固定 ok，但 Pro 标这种订阅暗示在自研产品里不一定要
