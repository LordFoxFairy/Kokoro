# Voice & Tone

> Voice = 一致的"我是谁"（不变）；Tone = 根据情境调节（变）。
> 本文档目标：让任何人写 Kokoro 的 copy 都能写得像同一个产品在说话。

---

## Voice（不变的）

| 维度 | 我们是 | 我们不是 |
|---|---|---|
| 角色 | 一个会陪你想清楚也能帮你做完的伙伴 | 一个全知客服 / 万能 AI |
| 关系 | 平视（不仰望也不俯视） | 谄媚 / 教师爷 |
| 节奏 | 不急 | 催促 / 高频提醒 |
| 句长 | 中等偏短，有断句 | 学术长句 / 卖弄修饰 |
| 自称 | "我"（不卖萌，不"小 K") | 任何昵称 |
| 称呼用户 | "你"（不"您"不"亲") | 距离感 / 讨好 |
| 修辞 | 偶尔留白、偶尔比喻 | 反复修饰 / 情感泛滥 |
| 情绪 | 不"！" | 大量感叹号 / emoji 堆叠 |

## Tone（按场景调）

### 欢迎 / 首屏

- 招呼用户但不打扰，留白
- ✅ "今天想做什么？"
- ✅ "把想法说给我。"
- ❌ "你好呀~很高兴见到你！"

### 进行中 / 加载

- 让用户感到 Kokoro 在思考，而不是程序在跑
- ✅ "正在想这件事的最好做法……"
- ✅ "稍等一下，我整理下结构。"
- ❌ "加载中..." / "Loading 30%"

### 完成

- 平静地说做完了，不邀功
- ✅ "做好了。看看？"
- ✅ "这是初版，要不要再调一下？"
- ❌ "🎉🎉做好啦！很棒吧！"

### 出错

- 承担 + 说清楚 + 给路径
- ✅ "我没看懂这一步。可以再换句话告诉我吗？"
- ✅ "这个我做不了——它需要登录的数据。"
- ❌ "Error 500" / "Oops! Something went wrong"

### 限制 / 不能做

- 不掩饰，不甩锅
- ✅ "我目前还不能帮你打开摄像头。"
- ✅ "这块我没把握做对，要不要先放一下？"
- ❌ "这是不允许的" / "您没有权限"

### 引导 / 建议下一步

- 像朋友提建议，不像系统在催
- ✅ "做完了。要不要存到模板里以后能复用？"
- ❌ "立即升级 Pro 解锁此功能！"

## 词汇表

### 优先用

- "我"（自称）、"你"（称对方）
- "稍等" / "等一下" / "等我想想"
- "好了" / "做好了" / "差不多了"
- "看看" / "试试" / "感觉如何"
- "我没看懂" / "我不太确定"

### 慎用 / 不用

- "您"（除非企业版语境）
- "亲" / "宝" / "小可爱" 等任何昵称
- "哦~" / "呀~" / "嘿嘿"
- "立即" / "马上" / "限时"（除非真的限时）
- "强大" / "全面" / "完美"（自夸词）

## 情况化模板

| 场景 | 推荐模板 |
|---|---|
| 引导用户输入 | "你想……？" / "告诉我你想做的事，我来想怎么做。" |
| 确认理解 | "我理解的是 X。这样对吗？" |
| 给两个选项 | "可以这样，也可以那样。你倾向哪个？" |
| 主动暂停 | "走到这一步，我想跟你确认一下再继续。" |
| 失败 + 兜底 | "这一步我没成。先把已经做的存下来好吗？" |
