# 设置（Settings）

> 用户控制自己跟 Kokoro 关系的地方。透明、可改、可走。

---

## 一句话用途

让用户**任何时候都能改掉**关于自己 / 关于 Kokoro 的东西——账号、个性、显示、数据、隐私。

---

## ASCII 布局速写

```
┌──────┬──────────────────────────────────────────────────────────┐
│ Side │  设置                                                      │
│      │  ──                                                       │
│      │  [账号]                                                    │
│      │  [个性化]    ← 我的指令 + Kokoro 记住的                      │
│      │  [显示]      ← 暗色 / 字号                                  │
│      │  [数据]      ← 导出 / 清除                                  │
│      │  [隐私]                                                    │
│      │  [关于]                                                    │
│      │                                                          │
│      │  ────────                                                 │
│      │                                                          │
│      │  右侧主区：当前所选分区的具体表单                              │
│      │                                                          │
└──────┴──────────────────────────────────────────────────────────┘
```

- 左纵向 tab（或顶部横向），右主区表单
- 一栏布局也可（适合移动端），分区折叠
- 风格延续 Kokoro：暖色、宽留白、柔交互

---

## 关键组件

| 组件 | 说明 |
|---|---|
| 分区导航 | 6 个分区，左侧或顶部 |
| 表单字段 | 标签 + 控件 + 一行说明文字（说明用人话，不用术语） |
| 保存 | 自动保存 + 顶部细横条提示 `已保存`（自动消失） |
| 危险按钮 | 红/暖色，二次确认；遵循 circuit-breakers |
| 退出登录 | 账号区底部 |

## 六个分区

### 1. 账号

| 字段 | 说明 |
|---|---|
| 头像 | 上传 / 默认（Kokoro 配色生成的暖色字母头像） |
| 显示名 | 公开可见 |
| 邮箱 | 主账号，登录用 |
| 密码 / 第三方登录 | 改密码 / 绑定 |
| 订阅状态 | 当前 plan / 续费 / 管理（v2 才会有） |
| 退出登录 | 底部一个按钮 |

### 2. 个性化（最重要 / 最有 Kokoro 气质）

**两块并列**：

#### A. 我的指令（用户写给 Kokoro）

- 一个多行输入框，用户写给 Kokoro 的长期指令
- 占位提示：`告诉我你是谁、你在做什么、你希望我怎么帮你。我会记着。`
- 例子卡片可参考：
  - `我是个设计师，主要做 B 端 SaaS。回答时不用解释什么是 B 端。`
  - `我说话偏短，回答也短一点。`
  - `我在做"早起社群"项目，相关都默认这个上下文。`

#### B. Kokoro 记住的（系统提取的记忆）

- 列表展示 Kokoro 从对话里**自动记下来**的事实
- 每条可单独删除 / 全部清空
- 每条形如：
  - `你正在做一个叫 "早起社群" 的项目`（2 天前）
  - `你喜欢用 Notion 来管理日常`（上周）
- 顶部一行说明：`这些是我在跟你聊的时候慢慢记下的。你可以删掉任何一条。`

> 三层模型参考：layer 1 = 系统人格（不暴露） / layer 2 = 我的指令（A） / layer 3 = 自动记忆（B）。详见 [04-architecture/memory.md](../04-architecture/memory.md)（如未建则待建）。

### 3. 显示

| 字段 | 选项 |
|---|---|
| 主题 | 浅色 / 深色 / 跟随系统 |
| 字号 | 小 / 中 / 大 |
| 紧凑模式 | 开 / 关（默认关，符合"愿为留白牺牲密度"） |
| 语言 | 中文 / English（v2） |

### 4. 数据

| 字段 | 说明 |
|---|---|
| 导出我的数据 | 打包下载所有对话 + Canvas（JSON + 资源） |
| 清除特定对话 | 跳转到 [library.md](./library.md) |
| 清除所有 | **危险操作** —— 二次确认 modal，输入"清除"四个字才能继续 |
| 注销账号 | 终极操作，单独 7 天宽限期 |

### 5. 隐私

| 字段 | 说明 |
|---|---|
| 用对话改进模型 | 默认 关 / 用户主动开（明文说明） |
| 我的产物对搜索引擎可见 | 默认 关（即使分享了链接，也不被 SE 索引除非显式打开） |
| 谁能看到我的模板作者主页 | 公开 / 仅有链接 / 关闭 |
| Cookie 与追踪 | 必要 / 分析 / 营销（分开同意） |

### 6. 关于

- 版本号
- 服务条款 / 隐私政策 链接
- 反馈渠道
- 一句话签名：`Kokoro · 把想法做出来`（或更软的版本）
- 致谢 / 开源声明（如使用开源组件）

## 关键状态

### A. 默认（has-content）

- 进入显示"账号"分区
- 字段填满 / 默认值清楚

### B. 编辑中（不显式 save）

- 字段变更 → 短暂 debounce → 自动保存
- 顶部细横条 `已保存`，2s 自动消失

### C. 危险操作 modal

- "清除所有"："你要清掉所有的对话和产物。这一步做完就回不去了。"
  - 输入 `清除` 才能继续
- "注销账号"："注销后 7 天内可以反悔。7 天后所有数据消失。"

### D. Empty（首次进个性化）

- "我的指令" 空时占位文案：`告诉我你是谁、你在做什么、你希望我怎么帮你。我会记着。`
- "Kokoro 记住的" 空时：`我们才刚认识，还没记下什么。聊多了就有了。`

### E. Error（保存失败）

- 顶部一行：`没保存上。[重试]`
- 字段保留用户输入

### F. Loading（首次进入）

- 表单骨架 + 柔脉动

### G. First-time（首次访问个性化）

- "我的指令"块顶部一次性提示：
  - `这里写给我的话，我会一直记着。`
