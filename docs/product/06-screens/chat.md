# 对话视图（Chat）

> 用户与 Kokoro 持续来回的地方。Canvas 未开时占主区域全部宽度，Canvas 开后缩到左半。

---

## 一句话用途

让对话**像在聊天**而不是在用工具：用户气泡右对齐（业界标准 IM 范式，零认知摩擦）、Kokoro 不带气泡（保留"像一封信"的叙述流差异化）、工具调用是可选展开的脚注，而不是噪音。详见 [ADR-008](../../decisions/ADR-008-chat-alignment.md)。

- 用户气泡：淡米色填充，大圆角，**右对齐**（IM 范式 — ChatGPT / Claude / WeChat / iMessage 都这样；详见 [ADR-008](../../decisions/ADR-008-chat-alignment.md)）
- Kokoro 回应：**无气泡**，纯文本左对齐（保留"像一封信"的叙述流差异化）
- 输入框始终钉在底部，与 Home 同款胶囊
- 输入框内：[🎨] = 切到 Canvas（提议/激活），[+] = 附件/工具，[↑] = 发送

## 关键组件

| 组件 | 说明 |
|---|---|
| 用户气泡 | 右对齐、淡米色填充、不对称圆角（右下角小、其余大）— IM 范式 |
| Kokoro 文本块 | 无气泡、左对齐、段落宽松 |
| 工具调用摘要 | 折叠态：图标 + 一行描述 + ▾。展开：参数 / 结果 / 时长 |
| 资料引用卡片 | 标题 / 来源域名 / 摘录 / 跳转 |
| Plan mode 提议块 | 步骤列表 + [Approve] [Refine] [Cancel] |
| 输入框 | 常驻底部，胶囊；含 Canvas 切换按钮 |
| 滚动锚定 | 新消息出现时自动滚到底，用户上滚后停滞 |

## 关键状态

### A. 对话中（has-content）

- 消息流正常滚动
- 输入框焦点保留，可随时打字
- 新 Kokoro 回应**流式输出**，光标位置脉动

### B. Kokoro 思考中（loading）

- 文案区域出现一行：`正在想这件事的最好做法……`
- 旁边一个**柔脉动小点**（不是 spinner）
- 输入框不禁用，用户可继续补话（视为"插话"）

### C. 工具调用进行中

- 在消息流里插入一个**折叠摘要块**：
  - `🔍 正在搜索"XXX"……` → 完成后 `🔍 搜索了 3 个来源 ▾`
  - `📄 正在读这个文件……`
- 默认折叠，点击展开看到调用参数 + 原始结果 + 耗时

### D. Plan mode 提议中

- 块样式：淡米色背景，左侧一道暖色细线
- 内容：
  ```
  我想这样做：
  1. ……
  2. ……
  3. ……
  [✓ 这样做]  [改一下]  [先不做]
  ```
- 用户不点之前，下面不会自动执行
- 详见 [core-flows.md](../03-product-form/core-flows.md#a-触发-plan-mode)

### E. 错误回复

- Kokoro 文本块照常出现，但带极淡边线
- copy 形如：`我没看懂这一步。可以再换句话告诉我吗？`
- 不显示 stack trace、错误码
- 严重错误（服务端 5xx）：底部出现一行小字 `连接出了点问题，[重试] 或 [稍后]`

### F. First-time（对话刚从 Home 跳入第一条）

- Home 的引导语**淡出移除**，第一条用户气泡淡入到中央
- 不要硬切换，要 200-300ms 自然过渡

### G. Empty（罕见，刷新历史空对话）

- 几乎不会出现；如出现，回到 Home 状态

## 关键 copy 示例

### 思考中

- `正在想这件事的最好做法……`
- `稍等一下，我整理下结构。`

### 工具调用摘要

- 搜索：`🔍 搜索了 3 个来源`
- 文件：`📄 读了你给的 PDF`
- 数据：`📊 跑了一下数据`

### 完成（涉及 Canvas）

- `做好了。看看？` （旁边 Canvas 缩略卡片）
- `这是初版，要不要再调一下？`

### Plan mode 提议

- `走到这一步，我想跟你确认一下再继续。`
- `我想这样做：……  你看可以吗？`

### 出错

- `我没看懂这一步。可以再换句话告诉我吗？`
- `这个我做不了——它需要登录的数据。`
- `连接出了点问题，[重试] 或 [稍后]`

### 限制

- `这块我没把握做对，要不要先放一下？`

## 与 Canvas 的过渡

- 用户点输入框内的 🎨 按钮 → Chat 收窄到 50%、Canvas 从右侧滑入
- 或 Kokoro 在回应中提议："要不要把这个落到 Canvas 上？[好] [先聊聊]"
- 过渡时长 ≈ 400-500ms，缓动 ease-out
- 详见 [canvas.md](./canvas.md#与-chat-的过渡)
