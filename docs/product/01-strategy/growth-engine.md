# 增长引擎

> 详细机制拆解见 [07-growth/](../07-growth/)，本文件是高层引擎说明。

## 一句话

**"产物 → 印记 → 看见 → 来用 → 模板 → 更多产物"——双循环互锁。**

## 双循环示意

```
            ┌─────────────────────────────────────────────┐
            │                                             │
            │            ① 产物即广告 循环                  │
            │                                             │
            │  用户做产物 → 产物带印记 → 分享 → 朋友看到    │
            │       ↑                                ↓    │
            │       └──── 朋友来用 Kokoro ────────────┘    │
            │                                             │
            └─────────────────────────────────────────────┘
                            ↓ ↑（互喂）
            ┌─────────────────────────────────────────────┐
            │                                             │
            │            ② 模板沉淀 循环                    │
            │                                             │
            │  产物一键转模板 → 社区模板库 → 新用户用模板  │
            │       ↑                                ↓    │
            │       └──── 更多产物 ─────────────────────┘  │
            │                                             │
            └─────────────────────────────────────────────┘
```

## 引擎成立的 3 个必要条件

1. **Canvas 产物本身有价值**——否则没人会分享 / 没人会用模板
2. **气质印记一眼可识别**——否则分享的是"内容"而不是"Kokoro 出品"
3. **分享 & 模板是一等公民**——产品出生第一天就具备，不是 v2 才加

任何一条不满足，引擎转不起来。

## 与"心"人格的耦合

气质印记 = 心路线的视觉具象化。所以 [02-personality/visual-language.md](../02-personality/visual-language.md) 与本引擎深度耦合：

- 默认 Canvas 模板必须自带印记
- 分享出去的产物必须带可识别的设计语言
- 用户即使改了内容，骨架/字体/色板的"心味"仍可识别

## 反面警示

- ❌ 把分享当成 v2 功能 — 引擎根本起不来
- ❌ 让用户能完全删掉 Kokoro 印记 — 产物即广告失效
- ❌ 把模板做成单独菜单 — 应该是"产物 → 一键转模板"的自然路径
- ❌ 试图给 Canvas 加"白标"模式（让企业去除 Kokoro 印记）— 跟增长引擎对立
