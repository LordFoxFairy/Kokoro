# 差异化

## 三个"凭什么不一样"

### 1. 气质护城河（核心）

通用 AI agent 红海里，**气质是最难被复制的资产**。功能可以追，"心"路线追不进来：

- Manus / ChatGPT / Gemini 都已经定型为中性专业，转温柔成本巨大
- 一旦"Kokoro = 温柔" 心智占住，新进者无差异化空间

### 2. 产物即广告

竞品的产物大多"没有产品印记"——你看到一份 ChatGPT Canvas 文档，看不出是 ChatGPT。Kokoro 反过来：

- 默认模板、字体、颜色、空状态、署名……每个产物都"自带 Kokoro 气质"
- 分享一份 Kokoro 产物 = 一次免费广告
- 这反向影响产品决策：**默认气质不能藏，要主动外露**

### 3. 通用底盘，但叙事钩子化

通用 agent 本身反增长，Kokoro 解法：

- 底盘通用（什么都能做）
- 但**对外叙事用 3-5 个 Canvas 钩子撑场**（参考 Manus 首页"产出物 chip"）
- 用户记住的是"Kokoro 能做小红书风海报 / 能做课件 / 能做策划案"，而不是"能做任何事"

## 与竞品的差异化对照

| 竞品 | 它强 | Kokoro 差异化 |
|---|---|---|
| Manus | 通用 agent + 视觉产出成熟 | Kokoro 有人格 + 产物自带印记 |
| ChatGPT | 通用 chat + 用户基数 | Kokoro 主战场是 Canvas，不是 chat |
| Gemini | Google 生态 + 模型能力 | Kokoro 暖调 + 心范，区隔 Google |
| CoWork | Plan-then-Approve + Claude 能力 | Kokoro 不绑特定模型，web-first |
| v0 / Lovable | 视觉产出 + dev 体验 | Kokoro 不限 dev，目标更广 |
| Notion AI | 集成已有文档库 | Kokoro 不替代文档系统，做产物加工台 |
| Canva / Figma | 设计成熟 + 模板生态 | Kokoro = AI 一句话生成，不是手动设计 |

## 风险

- **气质护城河不堵漏**：竞品偷气质难，但**仿气质** 容易（皮肤、文案）。需要靠**深度一致性**而非装饰
- **通用陷阱**：如果钩子叙事没做好，依然会陷入"什么都能做 = 用户记不住"
