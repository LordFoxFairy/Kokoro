# 分享一等公民

> 「一等公民」的意思：分享**不是**做完之后的额外动作，而是产物形态的一部分。
> 出生第一天就具备，不是 v2 才加。这条不让步。

---

## 总形态

每个 Canvas 产物自带 5 件分享相关的东西：

1. 一条 **share link**（不需要登录可见）
2. 一个 **公开落地页**（带印记的展示页）
3. 一张 **OG 图**（社交媒体预览自动生成）
4. 一段 **embed 嵌入代码**（v2，先占位）
5. 一个 **可选下载**（PNG / PDF / 链接图卡）

---

## 1. Share Link 结构

### 1.1 双层 URL 设计

| 形态 | URL | 用途 |
|---|---|---|
| **短链**（口播友好） | `kokoro.app/c/<8 位短码>` | 朋友圈、口播、二维码 |
| **长链**（SEO 友好） | `kokoro.app/share/<canvas_slug>` | 公开落地页正式 URL，被搜索引擎收录 |

短链 302 → 长链。短码用 base62（小写优先，避免歧义字符 `1lI0Oo`），8 位 ≈ 200T 容量，单产物不复用。

理由：
- 口播友好 + SEO 友好不可兼得，分成两条
- 单层短链 SEO 不友好；单层长链朋友圈传不动
- 参考 bit.ly + 站内长链组合，但短链放在自有域，避免被墙 / 屏蔽

### 1.2 过期策略

| 产物类型 | 默认有效期 | 可延长 | 可永久 |
|---|---|---|---|
| 免费用户 · 默认 | 永久 | — | 是 |
| 免费用户 · 含敏感数据标记 | 30 天 | 否 | 否 |
| 用户主动设置 | 7 / 30 / 90 / 永久 / 自定义 | — | — |
| Pro · 密码保护链接 | 永久 | — | 是 |

不做"默认过期"——分享一等公民意味着默认就是耐用的资产。临时分享走"可选过期"。

### 1.3 链接元信息（hover / 长按预览）

- 产物标题、缩略图、创建者署名（如同意公开）、Kokoro 角标
- 不暴露：用户邮箱、IP、设备信息

## 2. 公开落地页设计

> 联动 [06-screens/share.md](../06-screens/share.md)（待写）。本文件先把信息架构定下来。

### 2.1 视觉骨架

```
┌──────────────────────────────────────────────┐
│  Kokoro 角标 ← 极淡                            │
├──────────────────────────────────────────────┤
│                                                │
│  ┌────────────────────────────────────────┐   │
│  │                                          │  │
│  │              产物主体                     │  │
│  │     （Canvas 渲染区，居中或全宽）         │  │
│  │                                          │  │
│  └────────────────────────────────────────┘   │
│                                                │
│  by <创作者署名>  · 用 Kokoro 做的             │
│                                                │
│  ┌────────────────────────────────────────┐   │
│  │  "想给自己也做一份吗？"                   │  │
│  │  [ 开始我的 Kokoro · ]                   │  │
│  └────────────────────────────────────────┘   │
│                                                │
│  ─ 同款模板（v2 才加，v1 先不显示）─           │
│                                                │
│  Footer：举报 · 隐私 · Kokoro                   │
└──────────────────────────────────────────────┘
```

### 2.2 决策点

| 决策 | 选择 | 理由 |
|---|---|---|
| 落地页要不要"内容站"化（推荐流、相关产物） | **不要** | 第一是会偏离"看产物"主路径；第二是 SEO 上做内容站会被 ChatGPT/Gemini 碾压 |
| 落地页要不要"水印"风的 Kokoro 大角标 | **不要大角标，要小印记** | 大角标 = 广告感强；印记藏在底色 / 字体 / 署名条 = 气质感 |
| 落地页要不要登录墙 | **不要** | 分享落地不能要求注册才能看，违反一等公民 |
| 是否允许"无署名"分享 | **允许"匿名 by 一位 Kokoro 用户"**，但不允许"无 Kokoro 痕迹" | 用户身份可匿，产品身份不可匿 |

## 3. OG 图自动生成模板

OG 图 = 产物分享到 IM / 社交媒体时的缩略大图（1200×630）。

### 3.1 模板分层

| 层 | 内容 | 谁负责 |
|---|---|---|
| 底层 | 产物本体的截图（自动 puppeteer / playwright 截 Canvas） | 系统自动 |
| 装饰层 | Kokoro 印记元素（角标、底色腰带、字体） | 默认模板 |
| 文字层 | 产物标题 + 创作者署名 | 模板渲染 |

### 3.2 模板风格

不要"标准 OG 模板"那种生硬 grid，按 Canvas 类型差异化：

- **小红书海报**：竖图直接当 OG 主体，左下角 Kokoro 小印记
- **落地页**：网页 mockup 风（顶部浏览器条 + 页面缩略），底部留 Kokoro 条
- **课件**：第一页 + Kokoro 印记
- **思维结构图**：结构图缩略 + 标题在上方

### 3.3 缓存

- 产物 ID hash → CDN
- 内容修改 → 重新生成
- 加版本号 query 防止社交平台缓存旧图（`?v=2`）

## 4. Embed (嵌入) v2 路线

v1 先**不做**，v2 加。但 v1 要在 share 页面留一个 "Embed (即将到来)" 占位，告诉用户这件事在我们路线图上。

### 4.1 形态预想

- `<iframe src="kokoro.app/embed/<canvas_id>" />`
- 默认带 Kokoro 角标（不可隐藏）
- 三种尺寸预设（compact / medium / full）
- 嵌入页禁用：评论、修改、删除（只读）

### 4.2 v2 触发条件

- 月活产物分享数 > 某阈值（具体在 [metrics.md](./metrics.md)）
- 用户主动反馈"我要嵌入到我的 Notion / 博客 / 网站"

## 5. 防滥用

### 5.1 举报

- 落地页 footer 永远有「举报」入口
- 举报理由分类：垃圾内容 / 误导信息 / 侵权 / 违法 / 其它
- 举报后 24 小时内人工审，确认下架

### 5.2 抓取保护

- 公开 share 页可被搜索引擎收录（SEO 是增长资产），但限制爬虫频率（robots + rate limit）
- API 不暴露"批量列出所有公开产物"接口
- OG 图 hotlink 不限制（让分享传播最大化）

### 5.3 敏感内容自动检测

- 创建时：模型生成阶段就有第一道护栏（详见 [09-safety/](../09-safety/)）
- 分享时：发布到公开域名前过第二道（图像 NSFW / 文本敏感词）
- 落地后：举报兜底

## 6. Kokoro 印记的"强度档位" ⭐

> 这是增长引擎的核心决策。详见 [engine-overview.md 节点 B](./engine-overview.md#循环--产物即广告)。

### 6.1 三档设计

| 档位 | 形态 | 给谁 |
|---|---|---|
| **① 自然**（默认） | 底色 + 字体 + 右下角小角标 + 落地页底部"by Kokoro"条 | 所有免费用户默认 |
| **② 醒目** | 加额外的 Kokoro 署名 banner / 颜色更明显 | 创作者主动选（"我想 show off 这是 Kokoro 做的"） |
| **③ 隐身** | 角标缩到最小（不可消失），字体/底色印记保留 | Pro 可解锁，但**不允许彻底去除** |

### 6.2 不可让步的红线

- 三档**都保留**：字体、骨架色板、署名条 这三件骨架级印记
- **没有第四档**（即「无印记」）。哪怕 Pro 用户也不行
- 商业版（如果未来做）也不卖"白标"——白标 = 增长引擎自杀（见 [business-model.md](../01-strategy/business-model.md) 反例清单）

### 6.3 这条为什么不让步

讨论这条时我反复问自己：「让 Pro 用户去掉印记，对增长损害大不大？」答案：极大。因为：
- 付费用户往往是高活跃创作者，他们的产物传播力最强
- 让他们的产物"不带印记"= 把最强广告位卖给了订阅费
- 等于 freemium 的免费层在补贴 Pro 层的"摆脱品牌"特权——逻辑上是反的
