# 增长引擎 · 细化拆解

> 高层引擎说明在 [01-strategy/growth-engine.md](../01-strategy/growth-engine.md)。本文件把"双循环"拆到**节点级**，每个节点对应一个产品决策。
> 写这份文档时我反复问自己同一句：每个箭头背后，到底是哪个产品决定在让它转？

---

## 这页文档想干什么

01-strategy 那篇是引擎的"宪法"——告诉你引擎为什么成立。
这一篇是引擎的"工程图"——告诉你每个节点该做什么产品决定，才能让引擎真正转起来。

阅读顺序：先看「双循环节点表」抓骨架，再看每条「触点 → 行为 → 数据 → 反馈」走具体决策。

---

## 双循环节点表（鸟瞰）

```
        ① 产物即广告 循环
   ┌─────────────────────────────────────────────────────────┐
   │ 节点 A 创作  →  节点 B 印记  →  节点 C 分享              │
   │      ↑                                  ↓              │
   │   节点 F                              节点 D 落地        │
   │   新用户来用                              ↓             │
   │      └─────────  节点 E 转化  ──────────┘              │
   └─────────────────────────────────────────────────────────┘
                          ↕
        ② 模板沉淀 循环
   ┌─────────────────────────────────────────────────────────┐
   │ 节点 G 一键转模板 → 节点 H 模板审核 → 节点 I 模板市场     │
   │      ↑                                       ↓          │
   │   节点 L                                  节点 J 使用    │
   │   产物再回流                                  ↓          │
   │      └────────────  节点 K 二次创作 ─────────┘          │
   └─────────────────────────────────────────────────────────┘

互锁点：J → A（用模板做新产物），D → F（看到产物的人成为新用户），
       K → G（基于模板的产物也可以再转模板）
```

---

## 循环 ①：产物即广告

### 四列表（触点 → 行为 → 数据 → 反馈）

| 节点 | 触点（用户能摸到的地方） | 行为（用户在做什么） | 数据（要追什么） | 反馈（成败回路） |
|---|---|---|---|---|
| **A · 创作** | Canvas 编辑面板 | 用户在心人格陪同下做产物 | 产物完成率、单产物耗时、AI 反复迭代次数 | 完成率低 → 检查 Canvas 类型钩子是否合适 |
| **B · 印记** | 默认模板自带印记，字体/底色/署名/角标 | 用户即使不改也已经"带着 Kokoro"出门 | 印记保留率（用户是否削弱了印记强度）、署名外露率 | 印记被削太多 → 看是不是默认档位不够"轻"，让人想反抗 |
| **C · 分享** | 一键分享按钮（链接 / 图片 / 嵌入 / 下载） | 用户主动把产物丢出去 | 分享触发率、分享渠道分布、分享成功率 | 触发率低 → 分享按钮是不是埋太深 |
| **D · 落地** | 公开 share 页（带 Kokoro 印记的产物展示） | 朋友点开链接看到产物 | 落地页 UV、停留时长、CTA 点击率 | 停留过短 → 落地页有没有"看完想自己也做一个"的钩子 |
| **E · 转化** | 落地页底部"我也想做一个" CTA → 主站首页 | 朋友转化为试用者 | 落地页 → 首页跳转率、试用注册率 | 转化弱 → CTA 文案 / 钩子 / 信任元素不够 |
| **F · 新用户来用** | 主站首屏，带 chip 钩子 | 新用户完成首次产物 | 首产物完成率、首产物分享率 | 首产物分享率高 → 循环正反馈成立 |

### 每个节点的产品决策

- **A · 创作**：要不要在产物完成时给一次「轻推」鼓励分享？（推：增长向；不推：克制向 / 避免油腻）→ 推荐"产物完成 → 自然过渡到分享卡"，不是弹窗。
- **B · 印记** ⭐ 关键决策点：**默认印记强度档位**。建议三档：①「自然」（轻角标 + 默认底色 + 字体）默认 ②「醒目」（明显署名 + 底部 Kokoro 条）创作者主动选 ③「隐身」（最小化但不可消失）Pro 可解锁。三档都不允许彻底去除印记。
- **C · 分享**：share link 长什么样？建议 `kokoro.app/c/<8 位短码>` 或 `share.kokoro.app/<短码>`。短码用 base62，8 位，方便口播；同时保留 OG 长 URL `kokoro.app/share/<canvas_id>`，公开页 SEO 用这一条，短链做 redirect。详见 [sharing-first-class.md](./sharing-first-class.md#share-link-结构)
- **D · 落地**：公开落地页是"只看产物"还是"产物 + 周边推荐"？建议 v1 只看产物，避免被当成内容站；v2 再上"同款模板"侧栏。
- **E · 转化**：CTA 文案。坚决不要"立即注册"这种功能性话术，用心人格："想给自己也做一份吗？"。
- **F · 新用户来用**：把分享来源的新用户**和首屏 chip 联动**——如果落地的是"小红书海报"产物，首页 chip 默认高亮"小红书海报"。

## 循环 ②：模板沉淀

### 四列表（触点 → 行为 → 数据 → 反馈）

| 节点 | 触点 | 行为 | 数据 | 反馈 |
|---|---|---|---|---|
| **G · 一键转模板** | 产物完成视图 / 库视图，"另存为模板"入口 | 用户把自己得意的产物结构化成模板 | 转模板触发率、转模板成功率、用户的转模板冷启动时间 | 触发率低 → 入口埋太深 / 用户不知道能复用 |
| **H · 模板审核** | 模板提交流 → 审核队列 | 系统 + 人工筛选 | 模板提交量、通过率、平均审核时长 | 通过率低 → 提交门槛要么太松要么标准没沟通好 |
| **I · 模板市场** | "灵感库" / "模板市场" 入口（位置 = 首屏 chip 兜底"更多……"展开） | 用户在浏览中被某个模板吸引 | 市场 UV、浏览深度、模板详情 → 使用按钮 CTR | 浏览深的人不点用 → 模板预览不够还原效果 |
| **J · 使用** | 模板 → "用这个开始"按钮 → 跳进 Canvas 编辑 | 用模板生成自己的产物 | 模板使用次数、单模板复用率、使用后完成率 | 用了不完成 → 模板对 AI 不够"友好" |
| **K · 二次创作** | Canvas 编辑面板（AI 在用模板做半成品再加工） | AI 把模板当 scaffolding，用户填血肉 | AI 编辑深度（diff 行数 / 区块替换数）、用户最终改动比例 | 用户全删了 → 模板适配性差 |
| **L · 产物再回流** | 二次创作产物完成 → 又回到节点 A | 又是一个新产物，可能再走循环 ① | 同节点 A | 由 ① 接管 |

### 每个节点的产品决策

- **G · 一键转模板**：触发位置。建议两处：①产物完成视图右上角 ②库视图右键菜单。**不放分享浮层里**——不要把分享和留存动作混在一起。
- **H · 模板审核** ⭐ 关键决策点：**开放还是策展？** 见 [templates-market.md](./templates-market.md)。建议 v1 走"自动过滤 + 人工抽检"，v2 看数据再决定是否开门。
- **I · 模板市场**：入口位置。建议"首页 chip 行 + 库视图侧栏 + 设置里的灵感库"三处入口，但**首页只通过 chip 的"更多……"展开承接**，不另起 tab，避免把"模板市场"包装成独立产品（参考 ChatGPT 的能力 grid 平铺）。
- **J · 使用**：模板预览要不要"实时渲染"还是"静态截图"？建议 v1 静态截图（成本低），v2 上 hover 预览。
- **K · 二次创作**：模板的"AI 适配性"——模板不只是"已经写好的内容"，而是"留好坑给 AI 填的骨架"。这是 Kokoro 模板和 Notion 模板/Canva 模板的本质区别。详见 [templates-market.md](./templates-market.md#模板的-ai-适配性)。

## 双循环互锁点（不互锁就是两个孤岛）

| 互锁 | 它在干什么 | 不互锁会怎样 |
|---|---|---|
| **J → A** | 用模板创建的产物会回到节点 A，再走循环 ① | 模板沉淀只是个静态库，没有传播 |
| **D → F** | 落地页访客被首页 chip 接住，看到自己刚刚被打动的那类产物 | 转化漏斗在 chip 那一步断掉 |
| **K → G** | 二次创作产物本身也能再转模板 | 模板只能由"原创"产生，沉淀速度慢 |

---

## 与"心"人格的耦合点

引擎不是脱离气质的工程图。每个节点都被「心」绑定：

- **节点 B（印记）**：印记 = 心的视觉具象。删掉印记 = 拆掉气质护城河。
- **节点 D（落地）**：落地页不只是"看产物"，是"被 Kokoro 氛围抱一下"。气质本身是转化要素。
- **节点 E（转化）**：CTA 文案的口气决定转化。心路线下，"立即注册" 是失分项。
- **节点 G/J（模板）**：模板默认带印记。模板复用次数越多，印记越深入用户脑子。
