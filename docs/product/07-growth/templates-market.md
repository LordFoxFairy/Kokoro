# 模板市场

> 模板沉淀循环（② 循环）的承载层。比 Manus Playbook 更轻，比 Notion 模板更"AI 友好"。
> 模板对 Kokoro 不只是冷启动工具——它是引擎的第二条腿。

---

## 形态先讲清楚

Kokoro 的模板**不是**：
- ❌ 一份预填好的产物文档（Notion 模板那种"现成内容"）
- ❌ 一组排版预设（Canva 模板那种"换文字就好"）

Kokoro 的模板**是**：
- ✅ 一个**留好坑给 AI 填的骨架**（结构 + 风格 + 提示词约束）
- ✅ "半成品 + AI 再生" 的组合，每次使用都生成不同内容

详见下面「[模板的 AI 适配性](#模板的-ai-适配性)」一节。

---

## 1. 浏览 / 提交 / 策展 三件套

### 1.1 浏览（消费侧）

- 入口：首页 chip 行的「更多……」展开 + 库视图侧栏 + 设置里"灵感库"
- **不**做独立顶导 tab——模板市场不是独立产品，是工作流的一部分
- 分类：按 [03-product-form/canvas-types.md](../03-product-form/canvas-types.md) 的产出物类型切分（小红书海报 / 落地页 / 课件 / 思维结构图 / ……）
- 排序：默认"近期热门"，可切"最新 / 复用最多 / 我收藏的"
- 缩略图：静态截图 v1，hover 动效预览 v2
- 详情页：缩略图大图 + 创作者署名 + 复用数 + 「用这个开始」主按钮 + 衍生产物画廊（看别人用这个做出了什么）

### 1.2 提交（生产侧）

- 入口：产物完成视图右上角"另存为模板" + 库视图右键
- **不**放在分享浮层里（不要混合分享和留存动作）
- 提交流：
  1. 命名 + 简介（≤140 字）
  2. 选分类 + 加 3-5 个标签
  3. 选缩略图（默认是产物首屏截图）
  4. 提示词骨架检查（让用户确认哪些字段会被 AI 填）
  5. 隐私确认（模板里的内容会公开吗）
  6. 提交

### 1.3 策展（运营侧）

- 后台审核队列
- "编辑精选"位（首页 chip 展开的前 6 个）
- 周榜 / 月榜
- 主动联系创作者做"专题"（v2）

## 2. ⭐ 关键决策：开放 vs 策展

> 这是 [growth-engine.md](../01-strategy/growth-engine.md) 的待答题之一。

### 选项对比

| 策略 | 长这样 | 优 | 劣 |
|---|---|---|---|
| **A. 完全开放** | 任何人提交，自动上架 | 沉淀速度快、社区感强 | 垃圾爆炸、质量稀释、品牌受损 |
| **B. 完全策展** | 提交后人工审，过审才上 | 质量保障 | 沉淀慢、运营重、规模上不去 |
| **C. 自动过滤 + 人工抽检**（推荐） | 自动过基本规则 → 默认上架（限缩可见性） → 人工抽检/置顶 | 平衡 | 抽检漏审风险 |

### Claude 推荐：C，分阶段

| 阶段 | 策略 | 出口 |
|---|---|---|
| **v1（0–1000 模板）** | 偏策展：人工审每一份，质量优先 | 模板数量上 1000 → 切 v2 |
| **v2（1000–10k）** | 自动过滤 + 抽检 + 编辑精选 | 复用率连续两月下降 → 切 v3 |
| **v3（10k+）** | 偏开放：自动上架 + 举报 + 算法降权 | — |

理由：早期信号噪比对气质保护至关重要，宁可慢；规模到了再换轨。

## 3. 审核机制（自动 + 人工）

### 3.1 自动层

- **基础规则**：标题/简介敏感词扫描、图片 NSFW 检测、URL 黑名单
- **重复检测**：模板骨架结构 hash → 相似度 > 阈值即标记重复
- **质量启发式**：标题长度、是否有缩略图、字段数是否合理（避免"空模板"）
- 未过自动层 → 拒绝 + 反馈具体原因

### 3.2 人工层

- 进入审核队列：自动层过了 + 抽检或首发用户
- 人工审 checklist：
  - 是否服务"心"路线（气质是否一致）
  - 是否真的"留好坑给 AI 填"，而不是预填死内容
  - 是否有商业广告 / 推广嫌疑
  - 是否抄袭明显存在的模板
- 审核结果：通过 / 拒绝 / 改进建议（回到提交者）

### 3.3 审核时长

- 目标：48 小时内给结果
- 超时自动转加急队列
- 通过率公开（季度报告）

## 4. 创作者激励

### 4.1 v1 三件套（不涉及现金）

| 激励 | 形态 | 触发 |
|---|---|---|
| **署名** | 模板详情页 + 衍生产物落地页 | 默认 |
| **复用数 badge** | 创作者主页展示 | 累积 |
| **入选编辑精选** | 首页 chip 展开 + 站内邮件通知 | 编辑选 |

### 4.2 v2（待商业模式锁定后再开）

- 复用数分成？（涉及 [business-model.md](../01-strategy/business-model.md)）
- 创作者订阅引流（用我的模板的人转 Pro，分一杯）
- 创作者主页 → 独立 URL（个人作品集）

### 4.3 不做的

- ❌ **现金购买模板**：会破坏分享一等公民的免费层兼容性
- ❌ **NFT / 数字藏品**：跟"心"气质冲突
- ❌ **排行榜竞赛**：会鼓励刷复用数

## 5. 反垃圾

### 5.1 重复 / 抄袭检测

- 提交时：模板骨架结构 hash 对比
- 上架后：定期跑相似度扫描
- 标记的处理：
  - 高相似（>90%）：合并提示（让两位创作者协商）
  - 中相似（70–90%）：保留两份但都不上精选
  - 低相似（<70%）：默认放行

### 5.2 SEO 灌水 / 推广嫌疑

- 标题 / 简介 / 标签里出现"营销关键词密度高"→ 自动标记
- 模板内容里有外链 / 联系方式 → 自动降权
- 累犯 → 封号

### 5.3 举报闭环

- 每个模板有举报入口
- 三次有效举报 → 临时下架待审
- 审过保留则恢复，否则永久下架

## 6. 模板的"AI 适配性" ⭐

> 这是 Kokoro 模板与 Notion / Canva 模板的本质区别。
> 没想清楚这一条之前，请别去做模板市场的 UI。

### 6.1 什么叫"留坑给 AI 填"

举个具体例子（小红书海报模板）：

**Notion / Canva 模板的做法**（不要这样）：
```
标题：【种草】xxx 实测，真心推荐！
内容：xxx 真的太好用了，从早用到晚……
图片：[占位图]
```
用户买了模板 → 自己改文案 → 自己换图。

**Kokoro 模板的做法**（要这样）：
```
{骨架}
  标题位：钩子型 + 表情，10-15 字  ← AI 根据用户输入生成
  正文段：4-5 段，每段 1 个 emoji 起头，强主观体验  ← AI 生成
  图片位：3 张，竖图 4:5，统一风格  ← AI 选 / 生成
  尾巴：求互动 + 软引导  ← AI 生成

{风格约束}
  字体：< 小红书风手写体 + sans 主体 >
  色调：< 暖粉 / 米白 / 浅木 >
  气质：心人格的「自然」档（默认印记）

{用户输入}
  你要推荐什么？__________
  谁会用？__________
  最打动你的一点是？__________
```

用户用模板时：填三个空 → AI 接管骨架 + 风格约束 → 生成完整产物。

### 6.2 模板的三层结构

| 层 | 内容 | 谁定 |
|---|---|---|
| **骨架** | 字段名、字段类型、字段约束（长度 / 格式 / 必填） | 模板创作者 |
| **风格约束** | 字体、色板、气质档位（强 / 中 / 自然） | 模板创作者，预设可选 |
| **用户输入** | 提示词留白 + 上传 + 选项 | 使用者填 |

AI 根据三层合成 → 产物。

### 6.3 这意味着模板创作者的工作

不是"做一个好看的成品"——是"设计一个能生成好看产物的 jig"。
更接近 prompt engineering，而不是 graphic design。

提交流程要在这一步给充分引导（参考向导 / 在线示例 / "试用我的模板"按钮先让创作者自己看效果）。
