# 病毒循环 · 量化

> 想把"产物即广告"从 PPT 词变成可被产品决策驱动的指标，必须能算 K-factor。
> 本文件给一个数字草稿，让你感受 K>1 这件事到底现实不现实。
> ⚠ 所有数字均为**草稿假设**，等真数据来覆盖。

---

## 1. K-factor 速成

```
K = i × c
```

- `i` = 平均每位用户发出的邀请数（产物分享、模板复用、直接邀请之和）
- `c` = 邀请的平均转化率（看到的人有多少比例会真的来用）

**K > 1**：自然增长（病毒式）
**K = 1**：自维持
**K < 1**：得靠外部获客补血

延伸公式（含周期）：
```
增长 ≈ U₀ × K^(t/cycle_time)
```
- `cycle_time` = 一轮"产物 → 看到 → 来用"的周期，对 Kokoro 估约 3-7 天

---

## 2. Kokoro 的触点拆解

不像传统社交 app 只有"邀请"一根线，Kokoro 有 4 类触点同时贡献 `i`：

| 触点 | 描述 | `i` 贡献 | `c` 估值 |
|---|---|---|---|
| **T1 · 公开 share link** | 用户主动分享产物链接（朋友圈/IM/微博） | i₁ | c₁ |
| **T2 · OG 图传播** | 链接被转发时社交 feed 里出现的预览图（被动） | i₂ | c₂ |
| **T3 · 模板复用** | 模板市场里浏览者用了某个模板 → 落地页带原作者署名 | i₃ | c₃ |
| **T4 · 直接邀请** | "邀请朋友"按钮 / 邀请链接 | i₄ | c₄ |

总：`K = (i₁·c₁ + i₂·c₂ + i₃·c₃ + i₄·c₄)`

## 3. 数字草稿（极不靠谱，仅用来感受可达性）

### 3.1 假设的基准用户行为

| 假设 | 值 | 依据 |
|---|---|---|
| 一位活跃用户平均周产物数 | 2 份 | 类比 Manus 用户产出频率（保守估） |
| 产物→分享触发率 | 35% | 类比 Canva 模板产出后分享率（中性估） |
| share link 平均触达人数 | 8 人 | 朋友圈/IM 小群规模 |
| OG 图被动曝光乘数 | ×3 | 链接被转发 / feed 算法分发的额外触达 |
| 模板复用率 | 5%（用户的产物被收为模板 + 被复用） | 早期模板池小，估值偏乐观 |
| 直接邀请按钮触发率 | 8% | Slack / Notion 早期数据类比 |

### 3.2 触点级估算

每位活跃用户周贡献：

| 触点 | i（次/周/人） | c | i·c |
|---|---|---|---|
| T1 share link | 2 × 35% × 8 = **5.6** | 4%（链接点开 → 注册） | **0.224** |
| T2 OG 图被动 | 5.6 × 3 = **16.8** | 1.5%（看了 OG 没点 share 链接但搜来） | **0.252** |
| T3 模板复用 | 2 × 5% × 3（一个模板平均被 3 人用） × 1（衍生产物署名外露） = **0.3** | 6%（看到原作者署名 → 来用） | **0.018** |
| T4 直接邀请 | 1 × 8% × 5（邀请数）= **0.4** | 15%（朋友直接邀请转化率高） | **0.060** |
| **合计 K** | | | **≈ 0.554** |

K ≈ 0.55，**未到 1**。但这是早期、单循环、未叠加模板冷启动效应的估计。

### 3.3 调到 K=1 需要什么

哪几个杠杆最敏感？

| 杠杆 | 调整 | ΔK |
|---|---|---|
| 分享触发率 35% → 50% | 多做"完成 → 轻推分享"卡片 | +0.07 |
| OG 图被动 c 1.5% → 3% | OG 图设计上更"想点开" | +0.25 |
| 模板复用率 5% → 15% | 模板沉淀循环成熟（v2 之后） | +0.04 |
| 直接邀请转化 15% → 25% | 邀请页设计 + 邀请理由更强 | +0.04 |
| share link c 4% → 7% | 公开落地页 CTA 更强 / 信任元素更足 | +0.17 |

把 OG 图 c 拉一倍 + share link c 拉到 7% + 分享触发率拉到 50%：
K ≈ 0.55 + 0.25 + 0.17 + 0.07 ≈ **1.04** ← 刚过 1

**结论**：K=1 是可达的，但**杠杆主要在 share 链接落地页 + OG 图**，不在所谓的"邀请按钮"。

## 4. 周期时间 `cycle_time`

K 之外，传播速度由 `cycle_time` 决定：

| 阶段 | 时长 | 影响因素 |
|---|---|---|
| 用户做产物 → 分享 | 1-2 天 | 创作者养成节奏 |
| 分享 → 朋友看到 | 0.5-3 天 | 社交平台分发节奏 |
| 朋友看到 → 来用 | 0.5-2 天 | 落地页吸引力 + CTA 强度 |
| **合计 cycle_time** | **3-7 天** | — |

如果 cycle_time = 5 天，K = 1.04：
- 30 天内增长 ≈ 1.04^6 ≈ 1.27 倍（缓慢）
- 90 天内 ≈ 1.04^18 ≈ 2.03 倍
- 180 天内 ≈ 1.04^36 ≈ 4.10 倍

如果 K = 1.2：
- 180 天 ≈ 1.2^36 ≈ **708 倍**（指数爆发）

→ K 从 1.0 涨到 1.2 的回报远比从 0.5 涨到 1.0 大。**先到 1.0 不是终点，是起点**。

## 5. K>1 的最小条件清单

要让 K > 1 在 Kokoro 上成立，必须同时具备：

1. **产物完成率 > 50%**：用户做不出产物就没有分享源头
2. **印记保留率 > 80%**：分享的产物里看不出 Kokoro = 引擎漏气
3. **share link CTR > 5%**：公开落地页的钩子要够
4. **首产物分享率 > 30%**：新用户第一次做完就分享的比例
5. **首产物 → D1 留存 > 40%**：新用户分享但不留下 = 漏斗虚高
6. **OG 图点击率 > 2%**：被动曝光的转化

任何一条低于阈值 → 找对应触点重做，而不是去优化"邀请按钮"。

---

## 6. K-factor 的反面理解

K-factor 不是 KPI，是结构。

| 误区 | 正解 |
|---|---|
| 「K 上去 = 增长好」 | K 上去 = 自传播部分占比上去；DAU、留存、产物质量是底座 |
| 「邀请按钮做得猛，K 就上去」 | T4 是占比最低的触点；T1/T2 才是主战场 |
| 「K 不到 1 没意义」 | K=0.5 + 付费投放/SEO 一样能跑，只是不能裸跑 |
| 「K 越大越好」 | K 过大 = 增长不可控，新用户体验跟不上，质量崩盘 |

## 7. 测量方法（先把口径埋好）

> 详细指标定义见 [metrics.md](./metrics.md)。

每个触点都要可被独立测：

| 触点 | 埋点 |
|---|---|
| T1 share link | share_link_clicked + share_link_signup（带 share id 归因） |
| T2 OG 图 | og_image_impression（来源页 referrer）+ og_image_signup |
| T3 模板复用 | template_used + template_used_signup（带原作者 id 归因） |
| T4 直接邀请 | invite_link_clicked + invite_link_signup（邀请人 id 归因） |

每周自动算一次 K，分触点显示，不要只看合计。
