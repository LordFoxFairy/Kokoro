## 三个核心区

### 1. Sidebar（导航）

**结构**：

```
[Kokoro 品牌 / 折叠按钮]
─────────────────────
+ 新对话                  ← 主操作
🔍 搜索
🎨 模板库                 ← 增长玩法入口（一等公民）
📚 库 / 收藏
─────────────────────
最近
  - <对话标题 1>
  - <对话标题 2>
  - ...
─────────────────────
[用户头像 / 设置]
```

**与 Gemini 差异**：
- Kokoro 把"模板库"放到一级入口（增长玩法）
- Kokoro 不强放"笔记本""Gem"这种 Google 专属概念

### 2. Chat 区（主舞台）

**默认态**（Canvas 未启动）：
- 大字引导语 + 输入框（中央偏上）
- 输入框下方：产出物 chip × 5（见 [canvas-types.md](../03-product-form/canvas-types.md)）
- 已有对话历史：消息流（用户气泡 + Kokoro 回应）

**Chat → Canvas 触发**：
- 用户主动点 Canvas 按钮（输入框内）
- 或 Kokoro 判断"这件事最好在 Canvas 做"，提议触发

### 3. Canvas 区（引擎）

**结构**：
- 顶部工具栏（项目名 / 操作 / 关闭）
- 主内容区（文档 / 海报 / 课件 ……）
- 右缘工具条（折叠态：3-5 个图标）

**Canvas 态下**：
- Chat 缩窄到约 40%（min 360px）
- Canvas 占约 60%（min 420px）— grid `minmax(360px, 40fr) minmax(420px, 60fr)`
- Sidebar 自动收窄到 240px（参考 Gemini）
- 详见 [06-screens/canvas.md](../06-screens/canvas.md)
