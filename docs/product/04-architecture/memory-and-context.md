# 记忆与上下文（Memory & Context）

> 这个文件回答：Kokoro 怎么"记住"用户？什么是这一次对话内的事，什么会跨对话留下？
> 灵感来源：Claude Code 的三层模型（详见 [research/claude-code/learnings/01-philosophy.md](../../research/claude-code/learnings/01-philosophy.md) 启示 2）。
> 心路线注脚：记忆不是炫技。Kokoro 应该让用户**清楚地知道自己被记住了什么、能改、能删**——这是"内观"的一部分。

---

## 一句话模型

Kokoro 的记忆分**三层**：

```
┌─────────────────────────────────────────────────────────┐
│ 3. 持久层（跨 Session）                                  │
│    ├─ 用户写的指令（你告诉我的）                        │
│    └─ Kokoro 自学的（我从你身上学到的）                 │
└─────────────────────────────────────────────────────────┘
                          ↑ 每次新 session 启动时按需注入
┌─────────────────────────────────────────────────────────┐
│ 2. Session（一段可命名 / 可分支 / 可恢复的对话）         │
│    └─ 多个 turn 组成；可被 fork、rename、archive         │
└─────────────────────────────────────────────────────────┘
                          ↑ 包含
┌─────────────────────────────────────────────────────────┐
│ 1. Context Window（这一轮 Kokoro 实际能看到的）          │
│    └─ 历史消息 + Canvas 当前状态 + 指令 + 工具结果       │
└─────────────────────────────────────────────────────────┘
```

三层是嵌套的：context 是 session 的快照，session 启动时从持久层注入相关知识。

---

## 第 1 层 · Context Window（当下这一轮）

**定义**：Kokoro 这一次响应实际"看到"的全部内容。

**包含**：
- 当前 session 的历史消息（可能被压缩）
- 当前 Canvas 的内容（如果在 Canvas 态）
- 当前 session 的指令（user-level + project-level）
- 上一步工具结果（Web 抓取、文件读取等）

**用户可见的入口**（参考 Claude Code 的 `/context`）：
- 输入框旁边一个极淡的"上下文饱和度"指示（不是百分比数字，而是一条很细的环或线）
- Hover 显示明细："已包含 N 条历史 + Canvas X 字 + 指令 Y 条"

**自动管理**：
- 接近上限时优先丢弃"旧的工具结果"，再压缩"较早的对话"
- 压缩时用户可见提示："为了继续聊得顺，我先把前面整理成了摘要。"
- 压缩**永不丢失** Canvas 当前状态和持久层指令

**用户可控操作**：

| 操作 | 行为 |
|---|---|
| 重置上下文 | 清空 context，但 session 仍可在历史里看到 |
| 重点压缩 | 用户告诉 Kokoro "压缩时优先保留 X"，类似 Claude Code 的 `/compact [focus]` |
| 排除某段 | 选中一段历史 → "不要再读这段" |

**心路线注脚**：压缩文案不要说 "Context truncated"，要说 "我先把前面整理了一下"。

---

## 第 2 层 · Session（一段对话）

**定义**：一段有名字、有起止、可恢复的对话。Sidebar 「最近」里每一条 = 一个 session。

**生命周期**：

```
新建 → 进行中 → 暂停（关 tab）→ 恢复 → 继续 → 归档 / 删除
                    ↓
                  分支（fork）→ 衍生一个新 session，原 session 不动
```

**字段**（数据模型层面）：

| 字段 | 含义 |
|---|---|
| `id` | session 唯一标识 |
| `name` | 用户给的名字 / Kokoro 自动总结的标题 |
| `created_at` / `updated_at` | 时间戳 |
| `parent_id` | 若是 fork 出来的，指向源 session |
| `canvas_ids` | 关联的 Canvas（一对多） |
| `archived` | 是否归档 |
| `mode_at_start` | 起始时的 [Mode](./modes.md)（若做 Mode） |

**用户可见的操作**（在 sidebar「最近」每条上 hover 出现）：

| 操作 | 描述 |
|---|---|
| 重命名 | 改名字（默认 Kokoro 自动起，用户可改） |
| 分支 | 复制一份从某个点开始，原 session 保留——参考 Claude Code 的 `/branch` |
| 归档 | 不出现在「最近」，进库 |
| 删除 | 删除（永久；可加 7 天回收站） |

**Fork（分支）的意义**：
- 用户在 Canvas 里写了三版方向，想第四版试一个完全不同的——fork 出来不动原版
- 用户聊到一半想换个 Kokoro 风格，又怕弄丢现在的——fork 一份去试
- 对应 Claude Code 启示 5 "对话是可分支的" + Kokoro 增长玩法（每个 fork 的 Canvas 也是一个新模板候选）

**心路线注脚**：fork 的 UI 用词不叫 "Branch"，叫"另起一支"或"从这里再走一遍"——更柔，且贴近"内观"叙事。

## 第 3 层 · 持久层（跨 Session）

这一层最关键，也最容易被产品做糊。**必须分两区，不混**。

### 3a · 用户写的指令（你告诉我的）

**对应 Claude Code 的 `CLAUDE.md`**——用户主动写下，明确希望 Kokoro 永远遵守。

**典型内容**：
- "我习惯写日报用列表不用段落"
- "我做的 Canvas 默认要带我的名字签名"
- "解释技术概念时用类比，不要直接抛术语"
- "我做设计相关的事时，参考的风格是 Notion / Things"

**层级**（参考 Claude Code 多层加载顺序，由广到窄）：

| 层级 | 例子 | 范围 |
|---|---|---|
| **User-level** | "我喜欢简短回复" | 所有 session 默认遵守 |
| **Workspace-level**（可选） | "在工作 workspace 里用更正式的语气" | workspace 内 session |
| **Session-level** | "这次帮我做毕业海报，风格走小红书" | 仅这一个 session |

**UI 设计**：
- 设置页有「我的指令」区，用户可读可写可删
- 每条指令右侧标明触发范围（全局 / workspace / 此 session）
- 写完不需要"保存"——按回车即生效，柔交互

### 3b · Kokoro 自学的（我从你身上学到的）

**对应 Claude Code 的 auto memory（`MEMORY.md`）**——Kokoro 在与用户交互过程中自己提炼的。

**典型内容**：
- "用户更喜欢用 emoji 表达情绪"（从历史观察出）
- "用户改过 3 次我的措辞，把'您'改成'你'"
- "用户的 Canvas 偏好暖色调"

**关键原则**——这区**必须用户可审、可删、可禁**：

| 原则 | 做法 |
|---|---|
| 透明 | 每条学习都写明"我是从哪次对话学到的" |
| 可拒 | 用户可以"删除这条"或"以后别再学这类东西" |
| 不偷偷扩展 | Kokoro 学新东西时主动提示："我注意到你 X，要不要我以后默认 Y？" |
| 可暂停 | 用户可关闭"自学"，只用「我写的指令」 |

**心路线注脚**：这区是 Kokoro 表达"我在认真看你"的最强场域。但**任何"我学到了……"的提示都不能像中老年微信表情包那样邀功**。语气参考 [voice-and-tone.md](../02-personality/voice-and-tone.md) 的"完成"场景：平静、不卖萌。

### 用户设置页布局

两区**必须分开**，不要合并成一个"个性化"标签：

```
设置 → 记忆
├── 我告诉 Kokoro 的         （用户写的指令）
│     [+ 新增]
│     · 我喜欢简短回复 [全局] [删]
│     · 解释技术用类比 [全局] [删]
│     · ……
│
└── Kokoro 记住的我          （AI 自学的，开关可关）
      开关：[●] 允许 Kokoro 从我的对话学习
      · 你常用 emoji 表达情绪      [从 2026-05-18 的对话] [删除] [别再学这类]
      · 你的 Canvas 偏暖色         [从 4 次对话观察] [删除]
      · ……
```

混合两区的失败案例：ChatGPT 早期把"自定义指令"和"saved memory"塞一起，用户搞不清自己写的和 AI 学的差异。Kokoro 不能犯这个错。

## 三层之间的关系

| 行为 | 影响哪一层 |
|---|---|
| 用户说一句话 | Context Window（这一轮） |
| 用户开新对话 | 新 Session（仍读持久层） |
| 用户 fork 对话 | 新 Session（复制了部分 context） |
| 用户在设置写「我喜欢 X」 | 持久层 3a（用户写的） |
| Kokoro 在对话中说"我以后默认 X 好吗？"用户同意 | 持久层 3b（AI 学的） |
| 用户 `/compact` | Context Window（不动 Session） |
| 用户归档对话 | Session 状态变化（持久层不动） |
