# Sidebar 导航规格

> 这是 [ia.md](./ia.md) 里 Sidebar 草图的展开。每一区段写清楚：放什么、什么状态、点了会怎样、移动端怎么塌。
> 心路线下的导航气质：不堆按钮，留白比图标重要，分组依靠空白而不是横线。

---

## 总体规格

| 项 | 值 |
|---|---|
| 宽度（默认） | 280px |
| 宽度（Canvas 态自动收窄） | 240px |
| 宽度（折叠） | 60px（仅图标） |
| 顶部到底部顺序 | 品牌 → 主操作 → 增长入口 → 个人内容 → 最近 → 用户区 |
| 分组方式 | 留白 + 极淡分隔线（非粗线、非颜色块） |
| 滚动 | 「最近」区独立滚动，上下其他区固定 |

---

## 区段一 · 品牌 / 折叠按钮

**内容**：
- 左：Kokoro 文字标（小，不抢眼）
- 右：折叠/展开按钮

**状态**：
- 默认 hover 才显折叠按钮
- 折叠态下品牌字隐藏，只留一个小标识占位

**心路线注脚**：品牌字用 serif 或 hand-drawn 点缀，不用粗体 sans 砸脸。

---

## 区段二 · 主操作（一等公民）

**内容**：

| 项 | 图标 | 行为 |
|---|---|---|
| 新对话 | + | 立刻进首屏空状态，焦点落在输入框 |
| 搜索 | 🔍 | 打开搜索浮层（不跳页），全文搜对话 + Canvas + 模板 |

**状态**：
- "新对话" 是视觉重点，可微强调（轻底色或字重）
- 快捷键：⌘N / ⌘K（参考 Gemini / ChatGPT 习惯）

**为什么不放在顶栏**：Kokoro 没有顶栏。顶栏一旦出现就会拉扯气质（看 Gemini 的极薄顶栏也只放无关项）。

---

## 区段三 · 模板库（增长玩法的一等入口）

**内容**：

| 项 | 图标 | 行为 |
|---|---|---|
| 模板库 | 🎨 | 跳到模板浏览页（独立路由） |

**为什么是一级入口而不是埋在"库"里**：
- Kokoro 的增长引擎之一是「Canvas 产物 → 模板沉淀 → 别人来用」（详见 [07-growth/](../07-growth/) 和 [00-overview/one-pager.md](../00-overview/one-pager.md) 第三条）
- 模板库不只是"我自己存的模板"，也包含"社区/官方/被分享给我的"
- 放二级会让"分享模板"这个增长动作摩擦变高，违背"分享是一等公民"原则

**状态**：
- 新模板（社区或官方有更新）时旁边出现极小标记（一个圆点，不数字气泡）
- Canvas 态下仍可见，不折叠

---

## 区段四 · 库 / 收藏（个人内容）

**内容**：

| 项 | 图标 | 行为 |
|---|---|---|
| 库 | 📚 | 跳到库页面：收藏的对话 / Canvas / 文件 / 自存模板 |

**与"模板库"的区别**：
- 模板库 = 可复用的产物结构（公共属性）
- 库 = 我自己存的东西（私人属性）

**状态**：
- 第一次有内容才显示"未读/新归档"提示
- 空时仍保留入口，不隐藏，避免用户找不到

---

## 区段五 · 最近（对话流）

**内容**：
- 最近 N 条对话标题（N 视窗高决定，无限滚动）
- 每条：标题 + 极淡的相对时间（"昨天" / "上周"）
- Hover 出操作小菜单（重命名 / 分支 / 归档 / 删除）

**状态**：

| 状态 | 表现 |
|---|---|
| 默认 | 文字标题，无图标 |
| 当前会话 | 极淡底色高亮（不加粗、不变色） |
| 流式生成中 | 标题右侧一个微脉动点（心路线的"呼吸"动效，不是 spinner） |
| 已分支（forked） | 标题前一个极小"↳"标记 |
| 归档 | 默认不出现在最近里，需进库查看 |

**分组**（参考 ChatGPT / Claude）：
- 今天
- 昨天
- 过去 7 天
- 过去 30 天
- 更早（按月）

**与 Gemini 差异**：Gemini 的"最近"几乎没有可操作性。Kokoro 给"分支 / 归档 / 重命名"hover 菜单，对应 [memory-and-context.md](./memory-and-context.md) 里的 session 模型。

---

## 区段六 · 用户区（底部常驻）

**内容**：
- 用户头像 + 名字（折叠态只头像）
- 点开浮出小菜单：设置 / 切换账号 / 帮助 / 反馈 / 登出

**状态**：
- 常驻底部，不滚动
- "最近"区滚动到底也不挡用户区（用 sticky）

**待答题（在 ia.md 已列）**：用户区是常驻底部还是顶部下拉？
- 倾向**常驻底部**。底部锚定符合心路线的"沉静"，顶部下拉更适合工具感强的产品（Linear / Notion）。

---

## 折叠态规格

触发：
- 用户主动点折叠按钮
- 进入 Canvas 时自动从 280px → 240px（不折叠到 60px，只是缩窄）
- 用户主动可继续折叠到 60px 纯图标态

折叠态行为：
- 只显图标 + 用户头像
- Hover 整条出文字提示（tooltip 极淡，无阴影）
- "最近"区折叠后变成一个图标，点击展开浮层显示最近列表

## 移动端适配（草图）

> 详细见 [06-screens/](../06-screens/) 各页响应式说明。这里只给 sidebar 在移动端的塌缩策略。

```
┌──────────────────────────────────┐
│  ☰  Kokoro              👤        │  ← 顶部细栏（只移动端有）
├──────────────────────────────────┤
│                                  │
│       Chat / Canvas 主区         │
│                                  │
└──────────────────────────────────┘
```

**规则**：
- 默认隐藏 sidebar，顶部出现"☰"按钮
- 点 ☰ → 全屏覆盖式抽屉（左滑出），不是半遮罩
- 抽屉内部结构和桌面 sidebar 一致，但「最近」一定要可见（移动端用户的主诉求就是回到上一个对话）
- 用户头像保留在顶部右侧，方便单手够到（不放抽屉底部）

**断点**：
- ≥1024px：默认展开 sidebar
- 768-1024px：默认折叠（240px → 60px），可手动展开
- <768px：sidebar 转抽屉

## 待你拍板

- [ ] "新对话"按钮的样式：纯文本 / 极淡底色 / pill 按钮？（影响视觉重点的层级）
- [ ] 搜索是浮层还是独立页？倾向浮层（不打断当前对话）
- [ ] 「最近」分组阈值（今天/昨天/...）是否要可配置？倾向不配置，固定
- [ ] 移动端是否需要底栏（bottom nav）作为补充？倾向不需要，顶栏 + 抽屉够用
- [ ] 折叠态下"模板库"是否仍占一格？倾向占（强调一等公民地位）
