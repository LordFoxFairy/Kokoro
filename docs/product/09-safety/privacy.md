# 隐私与数据

> 隐私不是 footer 链接里那份合规文档，是产品决策。本文给"用户输入怎么用 / Canvas 归谁 / 分享给谁可见 / 何时走三方 / 怎么删"的产品答案。
> 心路线下的隐私语言：直白、不藏菜单、不"为了您更好的体验"那一套（参考 [voice-and-tone.md](../02-personality/voice-and-tone.md)）。

---

## 一句话立场

**用户输入和 Canvas 产物属于用户。Kokoro 默认不拿它们做训练。**

这不是营销话术，是产品默认值的必然结论——一旦默认值不是这样，后面所有"分享 / 模板 / 协作"的信任基础都垮。

---

## 1. 用户输入的用法

### 默认行为

| 数据 | Kokoro 怎么用 | 默认 |
|---|---|---|
| 你的对话内容 | 跑你这次任务、留作你自己的会话历史 | 必用 |
| 你的 Canvas | 同上 | 必用 |
| 你的对话用作模型训练 | 不用 | **opt-in**（不是 opt-out） |
| 你的对话用作 Kokoro 内部质量改进（人工审阅、对比模型输出） | 不用 | **opt-in** |
| 异常 / 错误日志 | 跑诊断、修 bug | 默认开，可关，不含 prompt 正文 |

> opt-in 的意思：开关默认 off，用户主动打开才算同意。不做"勾选默认勾上"。

### 用户控制台必须有的开关（在设置 → 隐私）

- 是否允许我的对话用于改进 Kokoro
- 是否允许我的 Canvas 模板用于官方推荐池
- 一键查看：当前我有哪些数据在 Kokoro

### 不做的事

- 不把对话内容卖给广告网络
- 不在对话里塞广告
- 不用对话内容做"个性化推荐"以外的画像（且推荐也只在 Kokoro 内闭环）

---

## 2. Canvas 产物归属

### 归属规则

| 主体 | 拥有 |
|---|---|
| 用户 | 你自己创建的 Canvas、模板、衍生作品 |
| Kokoro | 不拥有；只代为存储 + 渲染 |
| 协作者 | 被分享的人按你给的权限读 / 评 / 写，不获得所有权 |

### 衍生场景

- **你用 Kokoro 模板做了一份**：模板提供方有"来源标记"权利，没有内容所有权
- **你用了第三方 MCP 接入的数据**：数据归数据源所有，你产出的 Canvas 仍归你（但你要为合规使用数据负责）
- **AI 生成内容的著作权**：依各地法律不同。Kokoro 不主张对 AI 生成部分的著作权；用户层面的归属和商用权益参考各地 AIGC 法规（中国/欧盟/美国差异较大）

### 用户离开 Kokoro 时

- 全量导出：JSON + Markdown + 媒体文件打包
- 导出格式要"能在别处打开"，不能搞专有格式锁死
- 导出不需要订阅，是基本权利

## 3. 分享出去的可见性

Kokoro 走"分享一等公民"策略（见 [07-growth/](../07-growth/)），所以分享的可见性必须分级且默认安全。

### 三级可见性

| 级别 | 谁能看 | 默认 | 适用 |
|---|---|---|---|
| **私有**（private） | 仅你 | 新建 Canvas 默认 | 创作中、未完成 |
| **链接知**（unlisted） | 拿到链接的人 | 主动"分享给朋友"时 | 给特定人看，不公开 |
| **公开**（public） | 任何人 + 可被官方推荐池索引 | 主动"发布到模板库"时 | 想沉淀为模板被复用 |

### 关键规则

- **新建一律默认私有**，没有任何例外
- 公开过的能撤回：撤回后链接立刻失效，但**搜索引擎缓存 / 别人已复制的副本**无法被 Kokoro 追回——这点必须在公开前告诉用户
- 公开后能编辑，但有"快照保存"机制让浏览者看到的版本稳定（避免"原作者偷偷改坏了"）
- "链接知"模式可设过期时间（24h / 7d / 永久），默认永久

### 撤销分享的措辞（心路线）

> "链接已收回。已经打开过的人那边的页面会变成'这份不再公开了'。如果他们截图或保存过，那一份我帮不到了。"

不写："操作成功" / "您的隐私已得到保护"。

## 4. 第三方共享

### 什么时候 Kokoro 会把数据送出去

| 场景 | 送给谁 | 用户知情方式 |
|---|---|---|
| 跑底层大模型（Anthropic / OpenAI / 自建） | 模型提供方 | 注册时一次性告知 + 设置页可查 |
| 用户接入第三方 MCP server | 该 server 提供方 | 接入前显式审批，每次会话提示一次 |
| 用户主动用 Web 搜索 / WebFetch | 目标网站 | 工具调用面板可见 |
| 法律传唤 / 监管要求 | 司法机关 | 法律范围内告知用户（除非禁令） |

### 不做的事

- 不为商业目的把对话内容送给第三方
- 不为广告联盟接入数据
- 不在用户不知情的情况下接入新的模型提供方（哪怕只是"提升质量"）

### 模型提供方的选择

- 用户可在设置看到"当前对话由哪个模型回答"
- 企业版可以指定"只用 X 家模型"或"只用我们自部署的模型"
- 默认走 Kokoro 选定的混合策略（公开在 [99-references/](../99-references/) 或类似公共说明里）

## 5. 留存与删除

### 留存

| 数据 | 留存时长 |
|---|---|
| 对话历史 | 用户主动删除前永久（订阅停止 90 天后转归档，180 天后清理） |
| Canvas 内容 | 同上 |
| 日志（不含 prompt 正文） | 90 天 |
| 已删除项 | 30 天回收站，可恢复，过后真删 |
| 备份 | 加密备份 60 天，超期循环覆盖 |

### 删除流程

- **删单个对话 / Canvas**：列表里删 → 进回收站 → 30 天后清理
- **删账号**：设置 → 账号 → 删除账号 → 二次确认（参考 [circuit-breakers.md](./circuit-breakers.md)）→ 7 天冷静期（其间可撤回）→ 永久删除 + 邮件确认
- **批量删历史**：设置 → 隐私 → 一键清空对话 / Canvas，同样进回收站

### 已分享内容的删除特殊性

- 删除一份已经被公开模板化的 Canvas：模板下架，但已经被别人 fork 的副本仍归他们（这点公开时必须告知）
- 删除账号：你公开过的模板会被匿名化（保留产物但去除个人信息），用户可选"全部下架"或"匿名保留"

---

## 6. 导出

- 任何时间，任意账号层级，都能导出全部数据
- 格式：JSON（对话） + Markdown（Canvas） + 原始媒体文件 + 一份 README 说明结构
- 大账号支持分批 + 邮件链接下载
- 导出动作不打扰别人（不通知协作者"他导出了"）

---

## 7. 合规（GDPR / 后续补全）

本节先列原则，具体条文待法务（见 待拍板）。

- **GDPR 用户权利**：访问 / 更正 / 删除 / 可携 / 限制处理 / 反对 / 不被自动决策——产品功能上都要支持
- **CCPA（加州）**：知情 / 删除 / opt-out 销售。Kokoro 不卖数据，但"销售"在 CCPA 定义里包括"为对价交换"，所以要明确声明
- **中国《个人信息保护法》**：境内数据不出境 / 出境评估、敏感个信单独同意
- **未成年人**：参考各地最低年龄要求（GDPR 16 岁、COPPA 13 岁、中国 14 岁敏感个信单独同意）。Kokoro 默认 13+，敏感场景 16+
- **数据本地化**：企业版按客户区域提供 EU / US / 亚太节点

## 措辞清单（用在 UI 里）

| 场景 | 怎么说 | 不要说 |
|---|---|---|
| 公开前提示 | "公开后任何人都能看到这份，包括你不认识的人。撤回后能拿到链接的人就看不到了，但他们之前可能存过副本。" | "您的内容将被公开发布" |
| 删除账号前 | "我会保留你的数据 7 天，期间随时能回来。7 天后我会真的删干净，包括备份。" | "您的账号将被永久注销" |
| 接第三方 server | "这一步会让 [Notion] 看到你这段对话的内容。要继续吗？" | "授权第三方应用访问您的数据" |
| 导出完成 | "压缩包好了。里面有你的全部对话、Canvas 和媒体文件。" | "数据导出已完成，请点击下载" |
