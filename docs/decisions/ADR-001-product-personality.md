## 上下文

讨论到"主打增长玩法"环节，定位收敛为**"通用 agent + Manus 风格视觉产出"**。但这个定位与 Manus 本身重合，必须找差异化抓手。比较了五条候选路径（场景聚焦 / 本地优先 / 开源 / 品牌人格 / 分享一等公民），用户授权由 Claude 选定主轴。

最终选定**"品牌人格 + 分享一等公民"**为增长引擎的双驱动，本 ADR 锁定**品牌人格**这一支。

---

## 决策

Kokoro 走**「心」路线**：柔、温、内观，与产品名 `Kokoro`（日语「こころ」= 心 / 内心）的语义直接对位。

气质参考点（不是抄，是定调）：Notion / Things / Cron / Soft AI 类工具的"克制但有温度"，**不是**Lovable / v0 那种夸张鲜明性格，也**不是** Linear / Claude 那种纯专业极简。

---

## 理由

1. **名字本身就是钩子**：Kokoro = 心。这个语义不用 onboarding，用户第一次看到就有感觉。不用这个天然资产相当于浪费起手牌
2. **温柔 ≠ 不专业**：参考 Notion / Things，产品质感承载专业感，copy/视觉/微交互承载气质，两者不冲突
3. **与"视觉产物"契合度最高**：心路线有现成的视觉语言体系（暖调 / 柔圆角 / 自然元素 / 留白 / 手写点缀）。分享出去的 Canvas 产物**一眼可辨识 Kokoro 出品**，这是增长玩法的命脉
4. **传播圈层亲增长**：温柔治愈在中文互联网是显学（小红书 / B 站学习区 / 设计师 / 内容创作者）。这些圈层天然爱分享，是分享驱动增长的高燃料
5. **气质是最强护城河**：功能可以追，气质很难追。Manus 想转温柔难，OpenAI 没那个 DNA

---

## Tradeoff（必须告知）

- **欧美市场可能吃不开**：北美用户对"温柔治愈"AI 反应较弱（他们更吃"高效 / 个性 / 反叛"）。如果未来目标市场含强欧美比重，需重新评估
- **全产品一致性压力大**：所有 copy / 错误提示 / 加载文案 / 视觉细节都要服务气质。这是产品纪律
- **早期会有质疑**：被对比成"为啥不像 Cursor 那么硬核 / 不像 ChatGPT 那么万能"。靠产品质量顶住

---

## 否决项

| 候选 | 否决理由 |
|---|---|
| 夸张个性（v0 / Lovable / Suno 路线） | 与"通用 agent"定位冲突（通用需要包容多种创作意图，性格鲜明会挑剔用户） |
| 纯专业极简（Linear / Claude / Notion 纯版） | 没有差异化，输不会输但也赢不了 |

---

## 设计含义

| 维度 | 「心」路线下的形态 |
|---|---|
| 主色 | 暖调（米色 / 浅木 / 浅粉 / 苔绿 之一），**不走 Gemini 冷蓝** |
| 字体 | sans 主体 + serif 或 hand-drawn 点缀（标题或品牌字） |
| 微交互 | 柔慢动效，曲线缓动，避免 spring/弹性回弹 |
| 文案 voice | 第一人称、谦逊、有温度但不油腻；避免"亲""哦"过度卖萌 |
| Canvas 默认模板 | 自带温柔印记，用户产出物可一眼识别 |
| 错误态 | "我没看懂这个" 而不是 "Error 500" |
| 空状态 | 用引导语而非占位图，给人陪伴感 |
| 加载态 | 不用 spinner，用自然节奏的脉动 / 渐变 |

---

## 后果

- 所有后续设计（视觉、copy、交互）必须先过"是否服务心路线"这一关
- ADR-002 起的"目标用户 / 主打 Canvas 类型 / 视觉 token / Voice & Tone"都将以本 ADR 为前提
- 如未来 review 决定改人格，新开 ADR-001a 去 supersede 本文，保留历史
